Closure: Counter Example

Advanced Concept

What is Closure? A function that "remembers" its outer variables even after the outer function has returned.

function outer() { let count = 0; return function() { count++; return count; } }

📊 Counter Using Closure

0
💡 Each counter has its OWN independent state (closure magic!)

📝 How Closure Works:


// Outer function creates a private variable
function createCounter(initialValue = 0) {
    let count = initialValue;  // Private variable
    
    // Inner function "closes over" count
    return {
        increment: function() { 
            count++; 
            return count;
        },
        decrement: function() { 
            count--; 
            return count;
        },
        getValue: function() { 
            return count; 
        }
    };
}

// Each counter has its OWN independent count!
const counter1 = createCounter();
const counter2 = createCounter(100);

Scope Types

Fundamental

Scope determines where variables are accessible in your code.

🌍 Global Scope

Variables declared outside any function - accessible EVERYWHERE

// Declared outside any function
var globalVar = "I'm global";
let globalLet = "Also global";
const globalConst = "Global constant";

// Accessible anywhere in the code

🔧 Function Scope

Variables declared INSIDE a function - accessible ONLY within that function

function myFunction() {
    var functionScoped = "Only inside function";
    let alsoFunctionScoped = "Also only inside";
    // Cannot access outside this function!
}

📦 Block Scope

Variables declared with let/const inside {} - accessible ONLY within that block

if (true) {
    let blockScoped = "Only inside this block";
    const alsoBlockScoped = "Also block scoped";
    var notBlockScoped = "LEAKS outside!"; // var ignores blocks!
}
// Cannot access let/const variables here
// But 'notBlockScoped' is accessible!

Hoisting

Important!

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before code execution.

📌 var Hoisting

// What you WRITE:
console.log(myVar);  // undefined (NOT an error!)
var myVar = "Hello";

// How JavaScript READS it:
var myVar;           // Declaration hoisted to top
console.log(myVar);  // undefined
myVar = "Hello";     // Assignment stays put

🔒 let Hoisting

// What you WRITE:
console.log(myLet);  // ReferenceError!
let myLet = "Hello";

// How JavaScript READS it:
// let IS hoisted but NOT initialized
// Temporal Dead Zone (TDZ) - Cannot access until declaration

⚡ function Hoisting

// What you WRITE:
sayHello();  // Works! "Hello"

function sayHello() {
    console.log("Hello");
}

// Function declarations are COMPLETELY hoisted
// Can be called BEFORE they are defined!

🎨 Visual Explanation

Before Hoisting (What you write)

console.log(x);
var x = 5;

console.log(y);
let y = 10;

test();
function test() {
    console.log("hi");
}
⬇️ HOISTING ⬇️

After Hoisting (How JS runs)

var x; // hoisted
function test() { // hoisted
    console.log("hi");
}

console.log(x); // undefined
x = 5;

// let y is hoisted but in TDZ
console.log(y); // Error!
let y = 10;

test(); // "hi" - works!

📖 Key Takeaways

🔐 Closure

Function + its lexical environment = remembers outer variables

🌍 Global Scope

Accessible everywhere, avoid polluting it!

📦 Block Scope

Use let/const for block-level scoping

⚠️ Hoisting

Declarations are hoisted, but only function declarations are fully hoisted
